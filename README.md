
# Описание проекта

Система позволяет:

- создавать пользователей
- хранить баланс пользователей
- выполнять переводы между пользователями
- добавлять транзакции в очередь
- обрабатывать транзакции последовательно и конкурентно
- защищать изменение баланса от `data race`

## Версии🌿 Версии проекта

По условию задания каждая часть проекта должна храниться в отдельной ветке.

### v1 — Часть 1. Основы

В этой версии было реализовано:

структура User
поля ID, Name, Balance
метод Deposit
метод Withdraw
проверка на недостаток средств
создание нескольких пользователей
выполнение простых операций пополнения и снятия денег

### v2 — Часть 2. Транзакции и платёжная система

В этой версии было добавлено:

структура Transaction
структура PaymentSystem
хранение пользователей в map
хранение транзакций в slice
метод AddUser
метод AddTransaction
метод ProcessingTransactions
обработка переводов между пользователями
обработка ошибок, если пользователь не найден или не хватает средств


### v3 — Часть 3. Конкурентная обработка транзакций

В этой версии было добавлено:

канал chan Transaction
функция-воркер worker
несколько горутин для обработки транзакций
sync.WaitGroup
конкурентная обработка очереди транзакций


### v4 — Часть 4. Защита от гонок данных

В этой версии было добавлено:

sync.Mutex в структуру User
защита методов Deposit
защита методов Withdraw
безопасная работа с балансом при конкурентной обработке транзакций

Почему это было добавлено:
После появления нескольких горутин возник риск data race, так как баланс мог изменяться одновременно из разных потоков выполнения. Mutex нужен для безопасного доступа к данным.
